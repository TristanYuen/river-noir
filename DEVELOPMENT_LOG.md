## 已解决的问题

- 将本地模式与未来联机模式统一在 `GameTransport` 下，服务器接入时无需重写 UI。
- 客户端快照与完整牌局状态分离，联机后不会通过常规状态泄漏其他玩家底牌。
- 下注逻辑分离“待行动玩家”和“加注权锁定玩家”，覆盖短码加注后仍需补跟但不能再次加注的情况。
- 每轮结束时退还唯一最高投入者的未跟注筹码，避免生成无人有资格争夺的边池。
- 胜率计算放入 Web Worker，避免主线程卡顿。
- 视觉素材首版全部使用 CSS、Unicode 花色和代码组件，无需生成式位图；后续若确需位图，只使用 Image 2。
- 结算后清空玩家投入字段，牌局结果单独保留底池，修复筹码守恒重复计算。
- 使用 npm workspace 替代在当前 Windows 环境中反复损坏锁文件的 pnpm 11.9.0。
- 移动端行动栏改为四列弹性网格，避免最小内容宽度撑大牌桌。

## 关键约束

- 规则引擎保持纯 TypeScript，不依赖 React、浏览器 API 或具体服务器框架。
- 服务端必须负责权威洗牌、发牌、行动校验、计时和结算。
- 高风险规则必须有直接测试证据，特别关注短码 All-in、多层边池、Heads-up 和平分底池。
- 不降低完整规则、双语、高级感 UI、移动端、联机协议和上线交付范围。
